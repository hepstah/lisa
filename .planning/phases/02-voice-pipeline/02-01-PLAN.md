**Step 3: Create backend/lisa/services/stt_service.py.** This wraps the OpenAI Whisper API per D-10 and VOICE-03.

Define custom exceptions at module level:
- `class STTError(Exception): pass` -- base error for STT failures
- `class STTTimeoutError(STTError): pass` -- raised when STT exceeds timeout

Define `class STTService`:
- `__init__(self, api_key: str, model: str = "whisper-1", timeout: float = 3.0)`:
  - If `api_key` is empty string, raise `ValueError("OpenAI API key is required for STT service")` (per pitfall 4: validate at construction, not first call)
  - Store `self._client = openai.AsyncOpenAI(api_key=api_key, timeout=httpx.Timeout(timeout, connect=5.0))` -- the openai SDK accepts an httpx.Timeout object
  - Store `self._model = model`

- `async def transcribe(self, audio_bytes: bytes, format: str = "wav") -> str`:
  - Create `audio_file = io.BytesIO(audio_bytes)` then set `audio_file.name = f"audio.{format}"` (SDK needs .name attribute)
  - Call `transcript = await self._client.audio.transcriptions.create(model=self._model, file=audio_file, response_format="text")`
  - Return `transcript.strip()` if non-empty, otherwise raise `STTError("No speech detected in audio")`
  - Catch `openai.APITimeoutError` and re-raise as `STTTimeoutError("STT request timed out")` per D-17
  - Catch `openai.APIConnectionError` and re-raise as `STTError("Cannot reach speech recognition service")` per D-19
  - Catch `openai.APIError` as generic and re-raise as `STTError(f"STT failed: {e}")` 
  - Import `httpx` for the Timeout object, `io` for BytesIO, `openai` for the SDK

Define at module level:

- `class LLMTimeoutError(LLMError): pass` -- timeout specific

Define a typed dict or dataclass for the result:

```python
from dataclasses import dataclass
from typing import Optional

@dataclass
class DeviceIntent:
    device_id: str
    action: str  # "turn_on" or "turn_off"

```

Define `DEVICE_CONTROL_TOOL` dict at module level, exactly matching the research pattern:
```python
DEVICE_CONTROL_TOOL = {
    "name": "control_device",
    "description": "Control a smart home device by turning it on or off. Use this when the user asks to control a device. Only use device_ids from the provided device list.",
    "input_schema": {
        "type": "object",
        "properties": {
            "device_id": {
                "type": "string",
                "description": "The exact device_id from the available devices list",
            },
            "action": {
                "type": "string",
                "enum": ["turn_on", "turn_off"],
                "description": "The action to perform",
            },
            "confirmation": {
                "type": "string",
                "description": "A short spoken confirmation message for the user",
            },
        },
        "required": ["device_id", "action", "confirmation"],
    },
}
```

Define `class LLMIntentService`:
- `__init__(self, api_key: str, model: str = "claude-haiku-4-5", timeout: float = 3.0)`:
  - If `api_key` is empty string, raise `ValueError("Anthropic API key is required for LLM intent service")`
  - Store `self._client = anthropic.AsyncAnthropic(api_key=api_key, timeout=httpx.Timeout(timeout, connect=5.0))`
  - Store `self._model = model`

- `async def parse_intent(self, text: str, devices: list[dict]) -> DeviceIntent | None`:
  - `devices` is a list of dicts like `[{"device_id": "fake-lamp-1", "alias": "Bedroom Lamp", "is_on": false}]`
  - Build device_list string: join each device as `"- device_id: {d['device_id']}, alias: {d['alias']}, is_on: {d['is_on']}"`
  - Build system_prompt: `"You are Lisa, a smart home voice assistant. You control devices for the user. Available devices:\n{device_list}\n\nIf the user's request does not match a device control action, do NOT call the tool."`
  - Call `response = await self._client.messages.create(model=self._model, max_tokens=200, system=system_prompt, tools=[DEVICE_CONTROL_TOOL], tool_choice={"type": "auto"}, messages=[{"role": "user", "content": text}])`
  - IMPORTANT: Use `tool_choice: {"type": "auto"}` NOT forced tool_choice. Per research open question 4, auto allows Claude to NOT call the tool for non-device requests, which correctly handles unknown intents (D-20). When Claude does not call the tool, return None.
  - Iterate `response.content` blocks. If `block.type == "tool_use"` and `block.name == "control_device"`, extract `block.input` and return `DeviceIntent(device_id=block.input["device_id"], action=block.input["action"], confirmation=block.input["confirmation"])`
  - If no tool_use block found, return `None` (unknown intent)
  - Catch `anthropic.APITimeoutError` and re-raise as `LLMTimeoutError("LLM request timed out")` per D-18
  - Catch `anthropic.APIConnectionError` and re-raise as `LLMError("Cannot reach intent processing service")` per D-19
  - Catch `anthropic.APIError` and re-raise as `LLMError(f"Intent parsing failed: {e}")`
