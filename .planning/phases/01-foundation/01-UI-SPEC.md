# Phase 1 -- UI Design Contract

> Visual and interaction contract for the Lisa dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (recommended -- initialize before Phase 1 execution) |
| Preset | Default shadcn preset with dark mode, zinc neutral, Tailwind CSS v4 |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif` |

### shadcn Initialization

shadcn must be initialized before dashboard implementation begins. Run during the first plan of Phase 1 after the Vite + React + Tailwind scaffold exists:

```bash
npx shadcn@latest init
```

Select: dark mode via class strategy, zinc base color, CSS variables enabled.

**Required shadcn components for Phase 1:**

| Component | Used For |
|-----------|----------|
| button | Device toggle, text command submit, device config actions |
| card | DeviceCard wrapper, command history container |
| badge | Device status (on/off, reachable/unreachable), command result (success/error) |
| input | Text command input, device config IP/credentials fields |
| label | Form field labels in device config |
| table | Command history log |
| switch | Device on/off toggle (primary interaction) |
| dialog | Device configuration form, destructive confirmations |
| separator | Visual breaks between dashboard sections |
| sonner | Toast notifications for command results and connection changes |
| skeleton | Loading states for device cards and command history |

Install all Phase 1 components:
```bash
npx shadcn@latest add button card badge input label table switch dialog separator sonner skeleton
```

### Why shadcn

The RESEARCH.md "Don't Hand-Roll" table identifies custom WebSocket reconnect as the only justified custom client code. Toggle switches, cards, tables, form inputs, and status badges are standard UI primitives. shadcn provides accessible (Radix-based), unstyled-by-default components that compose with Tailwind v4. This avoids hand-rolling accessibility patterns (keyboard nav, screen reader labels, focus management) for the 11 components listed above.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Compact element spacing, card internal padding on mobile |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, card grid gap |
| xl | 32px | Page horizontal padding on desktop |
| 2xl | 48px | Major section vertical separation |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: Touch targets on toggle switches and buttons must be at minimum 44px (per WCAG 2.5.5) for reliable mobile use. Tailwind class: `min-h-11 min-w-11` (44px).

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight` |

**Font weights used:** 400 (body text, table content, timestamps) and 600 (headings, CTA labels, device names). Medium (500) is permitted only for form labels.

**Monospace:** Timestamps and error details use `font-mono text-xs` (12px monospace) for scanability in the command history table.

## Color

This dashboard uses a **dark theme** as the default and only theme for v1. Smart-home dashboards are frequently viewed in dim environments and a dark surface reduces visual strain. This also aligns with modern dashboard conventions.

shadcn's zinc palette with CSS variables provides the foundation. The specific values below map to shadcn's default dark mode tokens:

| Role | CSS Variable | Value (dark) | Usage |
|------|-------------|--------------|-------|
| Dominant (60%) | `--background` | zinc-950 (#09090b) | Page background, main surfaces |
| Secondary (30%) | `--card` | zinc-900 (#18181b) | Device cards, command history panel, config dialog |
| Accent (10%) | `--primary` | zinc-50 (#fafafa) | See reserved list below |
| Destructive | `--destructive` | red-500 (#ef4444) | Delete device, error badges only |

### Accent Reserved For

The accent color (zinc-50 as primary on dark background) is reserved for exactly these elements:

1. Device toggle switch (active/on state track color)
2. "Send" button on text command input
3. "Add Device" button in device config
4. "Save" button in configuration dialog
5. Active connection status dot (green variant -- see semantic colors below)

Accent must NOT be applied to: table headers, section titles, card borders, or decorative elements.

### Semantic Status Colors

| Status | Color | Tailwind | Used On |
|--------|-------|----------|---------|
| Device ON | emerald-500 (#10b981) | `text-emerald-500` | Device status badge, status dot |
| Device OFF | zinc-500 (#71717a) | `text-zinc-500` | Device status badge |
| Unreachable | amber-500 (#f59e0b) | `text-amber-500` | Device unreachable badge, connectivity warning |
| Error / Failed | red-500 (#ef4444) | `text-red-500` | Command failure badge, error messages |
| Success | emerald-500 (#10b981) | `text-emerald-500` | Command success badge |
| Connected (WS) | emerald-500 (#10b981) | `bg-emerald-500` | StatusBar connection dot |
| Disconnected (WS) | red-500 (#ef4444) | `bg-red-500` | StatusBar connection dot |
| Reconnecting (WS) | amber-500 (#f59e0b) | `bg-amber-500` | StatusBar connection dot (pulsing) |

## Layout

### Responsive Strategy

Mobile-first. Single column on mobile, two-column grid on tablet and above.

| Breakpoint | Layout | Behavior |
|------------|--------|----------|
| < 640px (mobile) | Single column, full width | Cards stack vertically, command history below devices |
| >= 640px (sm) | Single column, centered max-w-xl | Comfortable reading width |
| >= 1024px (lg) | Two columns: devices left (2/3), command panel right (1/3) | Side-by-side device grid and command history |

### DeviceCard Anatomy

```
+----------------------------------+
| [Icon]  Device Alias     [Toggle]|
|                                  |
| Status badge: ON / OFF           |
| Reachability: Reachable /        |
|               Unreachable        |
+----------------------------------+
```

- Card: `rounded-lg border bg-card p-md`
- Device alias: Heading style (14px semibold -- not the full heading size, use `text-sm font-semibold`)
- Toggle: shadcn Switch component, disabled when device is unreachable
- Status badge: shadcn Badge with semantic color variant
- Unreachable state: Toggle disabled, amber warning badge, card opacity reduced to 60%

| Column | Content | Width |
|--------|---------|-------|

## Interaction States

### Device Toggle

| State | Behavior |
|-------|----------|
| Idle (on) | Switch is in ON position, emerald track |
| Idle (off) | Switch is in OFF position, zinc track |
| Toggling | Switch shows loading indicator (brief opacity pulse), disabled during request |
| Success | Switch moves to new position, toast notification: "{alias} turned {on/off}" |
| Error | Switch reverts to previous position, toast notification with error message (destructive variant) |
| Unreachable | Switch disabled, reduced opacity, amber "Unreachable" badge shown |

### Text Command Input

| State | Behavior |
|-------|----------|
| Empty | Placeholder text: "Type a command..." |
| Typing | Standard input behavior, Send button enabled when input is non-empty |
| Submitting | Input disabled, Send button shows loading state (spinner icon replaces send icon) |
| Success | Input clears, command appears in history table with success badge |
| Error | Input retains text (user can edit and retry), toast shows error, command appears in history with error badge |

### WebSocket Connection

| State | StatusBar Indicator |
|-------|---------------------|
| Connected | Small emerald dot + "Connected" label (label hidden on mobile, dot only) |
| Disconnected | Small red dot + "Disconnected" label |
| Reconnecting | Small amber dot with CSS pulse animation + "Reconnecting..." label |

### Device Discovery (in config dialog)

| State | Behavior |
|-------|----------|
| Idle | "Discover Devices" button enabled |
| Discovering | Button shows spinner, text changes to "Scanning..." |
| Found devices | List of discovered devices appears below button, each with [Add] action |
| No devices found | Message: "No devices found on your network. Check that devices are powered on, or add one manually below." |
| Error | Message: "Discovery failed: {error}. You can add a device manually below." |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Toggle keyboard access | shadcn Switch handles Enter/Space natively (Radix) |
| Screen reader: device state | `aria-label="{alias}: {on/off}, {reachable/unreachable}"` on each DeviceCard |
| Screen reader: toggle | `aria-label="Toggle {alias}"` on Switch |
| Screen reader: status badges | Badge text is the accessible label (no icon-only badges) |
| Screen reader: command history | Table uses semantic `<table>` with `<th>` headers |
| Focus visible | Tailwind `focus-visible:ring-2 focus-visible:ring-ring` on all interactive elements (shadcn default) |
| Color contrast | All text on dark backgrounds meets WCAG AA (4.5:1 minimum). zinc-400 (#a1a1aa) on zinc-950 = 7.4:1. emerald-500 on zinc-950 = 5.1:1. |
| Touch targets | Minimum 44x44px for all interactive elements on mobile |
| Reduced motion | Respect `prefers-reduced-motion`: disable pulse animations, use instant transitions |
