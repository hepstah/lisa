Create deployment and configuration artifacts.

**systemd/lisa-backend.service** (per D-08, INFRA-02, per Research systemd example):
```ini
[Unit]
Description=Lisa Smart Home Backend
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/lisa/backend
ExecStart=/home/pi/lisa/backend/.venv/bin/uvicorn lisa.main:app --host 0.0.0.0 --port 8000
Restart=on-failure
RestartSec=5
Environment=LISA_DEV_MODE=false

[Install]
WantedBy=multi-user.target
```

**.env.example** (top-level, documents all configuration):
```
# Lisa Smart Home Configuration
# Copy to .env and edit values

# Development mode (true = use fake device adapter)
LISA_DEV_MODE=true

# SQLite database path
LISA_DB_PATH=lisa.db

# Backend server
LISA_HOST=0.0.0.0
LISA_PORT=8000

# TP-Link Kasa credentials (required for newer devices)
# Leave empty if your devices do not require authentication
LISA_KASA_USERNAME=
LISA_KASA_PASSWORD=
```

   - GET http://localhost:8000/api/devices/ returns JSON
   - GET http://localhost:8000/ serves the dashboard HTML (from dist/)

If the StaticFiles mount or Vite proxy is missing or incorrect, fix it. The backend's main.py should already contain both from Plan 02, but verify and correct if needed.
