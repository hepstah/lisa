    **Part A: Create backend/lisa/voice/voice_loop.py**

    Create the VoiceLoop class that runs a continuous blocking audio loop in a background thread. Follow the exact pattern from 03-RESEARCH.md Pattern 1.

    Class signature:
    ```python
    class VoiceLoop:
        def __init__(self, wake_detector, audio_capture, pipeline, event_loop, status_callback):
    ```

    Constructor stores all dependencies. `event_loop` is the main asyncio event loop (for bridging thread -> async). `status_callback` is an async callable that broadcasts pipeline status.

    Methods:
    - `start()`: sets `_running = True`, creates daemon thread calling `_run()`, starts it
    - `stop()`: sets `_running = False`, joins thread with timeout=5
    - `_emit_status(status: str)`: calls `asyncio.run_coroutine_threadsafe(self._status_cb(status), self._loop)`
    - `_run()`: the main blocking loop:
      1. Import pyaudio at runtime (graceful, Pi-only)
      2. Open PyAudio stream: format=paInt16, channels=1, rate=16000, input=True, frames_per_buffer=1280
      3. Emit "listening" status
      4. Loop while `_running`:
         a. Read 1280 frames from stream (exception_on_overflow=False)
         b. Call `self._wake.detect(frame)` -- if empty dict, continue
         c. On detection: log wake word, emit "processing" status
         d. Call `self._capture.reset()`
         e. Feed triggering frame via `self._capture.process_frame(frame)`
         f. Continue reading frames and feeding to `self._capture.process_frame()` until it returns False
         g. Get audio bytes via `self._capture.get_audio()`
         h. Mute wake detector: `self._wake.mute()`
         i. Call pipeline via `asyncio.run_coroutine_threadsafe(self._pipeline.process_audio(audio_bytes), self._loop)` and `.result(timeout=15)`
         j. Emit "responding" status (TTS playback happens inside pipeline)
         k. Sleep 0.5s cooldown (echo prevention per Pitfall 1)
         l. Unmute wake detector: `self._wake.unmute()`
         m. Emit "listening" status
      5. Wrap entire `_run()` body in try/except: on exception, log with logger.exception, emit "error" status
      6. Finally block: stop stream, close stream, terminate PyAudio

    Important: Use the graceful import pattern for pyaudio (try/except ImportError) at the module level, same as wake_word.py and tts_service.py. Set PYAUDIO_AVAILABLE flag. In `_run()`, raise RuntimeError if not available.

    Add this block AFTER `commands.voice_pipeline = voice_pipeline` (after line 85) and BEFORE the `yield`:

    ```python
    # Phase 3: Voice loop (Pi mode only)
    voice_loop = None
    if voice_pipeline and not settings.dev_mode:
        try:
            from lisa.voice.voice_loop import VoiceLoop
            from lisa.voice.wake_word import WakeWordDetector
            from lisa.voice.audio_capture import AudioCapture

            loop = asyncio.get_event_loop()

            async def pipeline_status_callback(status: str):
                await manager.broadcast({"type": "pipeline_status", "status": status})

            voice_loop = VoiceLoop(
                wake_detector=WakeWordDetector(),
                audio_capture=AudioCapture(),
                pipeline=voice_pipeline,
                event_loop=loop,
                status_callback=pipeline_status_callback,
            )
            voice_loop.start()
            logger.info("Voice loop started")
        except Exception as e:
            logger.warning("Voice loop unavailable: %s", e)
    else:
        if not voice_pipeline:
            logger.info("Voice loop skipped: voice pipeline not available")
        elif settings.dev_mode:
            logger.info("Voice loop skipped: dev mode (use /api/commands/text)")
    ```
